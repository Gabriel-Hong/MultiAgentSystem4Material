## Slide 3: Material DB란?

> "엔솔을 처음 접하는 분들도 계시니, 먼저 Material DB가 무엇인지 간단히 설명드리겠습니다."

### Material DB = 각 나라의 설계 기준에 맞는 재료 데이터베이스

- 각 국가마다 철근, 콘크리트, 강재 등의 **설계 기준이 다름**
- 엔솔이 해당 국가에서 사용되려면 **그 나라의 Material DB가 반드시 필요**
- 예: 한국(KS), 미국(ASTM), 유럽(EN), 중국(GB), 태국, 베트남 등

### 📌 핵심
**Material DB가 없으면 → 그 나라에서 엔솔을 쓸 수 없다**

## Slide 18 (쿠키): 트러블슈팅 #1 - 대용량 코드 처리

### ❓ 문제: 17,000줄 코드를 LLM에 어떻게 넣을까?

- GPT-4의 컨텍스트 윈도우 한계
- 토큰 비용 폭발

### ✅ 해결: Clang AST 압축

```
17,000줄 → 500줄 (97% 압축)
```

- 코드의 **구조만 추출**하여 LLM에 전달
- 불필요한 구현 세부사항 제거
- **핵심 패턴만** 분석하도록 최적화

### 📊 결과

| 처리 시간 | 38초 |

## Slide 20 (쿠키): 트러블슈팅 #3 - 안전한 자동화 (Guardrails)

### ❓ 문제: LLM이 잘못된 코드를 생성하면?

- **환각(Hallucination)** 위험
- 범위 외 요청 처리 위험
- 자동화 시스템의 신뢰성 확보 필요

### ✅ 해결: 3단계 검증 시스템

```
Layer 1: Confidence Score Validation
         └── Calculate Confidence (0.0 ~ 1.0)
         └── Threshold: 0.9
         └── Score >= 0.9? → Pass / Reject

Layer 2: Health Check
         └── GET /health
         └── Expected: Status 200
         └── Healthy? → Pass / Reject

Layer 3: Timeout Guard
         └── Max: 300s
         └── 초과 시 중단
```

### 📊 결과

| 지표 | 값 |
|------|-----|
| 통과율 | **98%** |
| 잘못된 처리 | **0건** |
| 신뢰도 기준 | ≥ 0.9 |

### 핵심 효과

| 효과 | 설명 |
|------|------|
| ✅ 범위 외 요청 | 자동 필터링 |
| 🧠 LLM 환각 | 사전 방지 |
| 🔒 안전한 자동화 | 품질 보장 |
